Deployment Roadmap

Domain-centric deployment quickly scales full capability in 6–9 months

Value every 16–20 weeks · Change management built-in
Phase 0
Planning, Architecture Design & MVP
  • Confirm tech stack and prioritize service areas
  • Define capability system layers
  • Design target architecture
  • MVP for 1–2 service areas, full human-in-the-loop
Release 1
MVP to Production
  • Deploy MVP to production
  • Stand up governance, thresholds, and guardrails
  • Establish AI Ops team and observability tooling
  • Validate against baseline metrics
Release 2
Expanded Domains
  • Expand to 2–3 additional service areas
  • Extend knowledge platform to new domains
  • Reduce human oversight progressively
  • Onboard model tuners and QA specialists
Release 3
Global Release
  • Unify intake and fulfillment across workflows
  • Deepen personalization
  • Continue legacy volume migration
  • Scale across all remaining service areas
  • Mature agent deployment and tuning model
  • Retire legacy queues and processes
Ongoing
Transition to Ops + Managed Services
  • Continuous tuning, refinement, and QA
  • Workforce transition support
  • Recalibrate onshore/offshore mix over time
Value drops every 16–20 weeks — with change management built into each release cycle, ensuring adoption keeps pace with capability.

Proof of Concept

What we need to deliver a POC tailor-made for Raptor

Four foundational inputs enable PwC to build and validate EVA in a non-production environment within weeks.

1
SME Availability
One support lead from Raptor (US) available for weekly reviews during the POC build — the single most critical input.
2
Sandbox Environment
Non-prod access to R6 Chat so we can build and test without touching live support infrastructure.
3
Knowledge Content
Existing FAQs, help articles, and agent resolution scripts to seed EVA's knowledge base with Raptor-specific intelligence.
4
Top Ticket Sample
500–1,000 real cases from the highest-volume issue types (login, StudentSafe, account access) to train and validate EVA.
